# 핵심 시스템

## 1. 몬스터

MEKI의 몬스터는 **Species(몬스터 원본 데이터)** 와 **Owned Monster(플레이어 보유 몬스터)** 로 구분하여 관리한다.

### Species (몬스터 원본 데이터)

Species는 모든 플레이어가 공유하는 몬스터의 기본 정보를 의미한다.

Species에는 다음 정보가 저장된다.

* 이름
* 지역
* 태생
* 도감 설명
* 아이콘
* 이미지 경로

Species에는 레벨이나 경험치 등의 플레이어 데이터를 저장하지 않는다.

---

### Owned Monster (플레이어 보유 몬스터)

Owned Monster는 플레이어가 실제로 보유한 몬스터를 의미한다.

플레이어 몬스터는 **개체(Instance) 단위**로 저장한다.

동일한 몬스터를 여러 마리 보유할 수 있으며,
각 몬스터는 고유한 monster_id를 가진다.

현재 저장 정보

* 소유자
* 몬스터 종류
* 획득일

향후 확장 예정

* 레벨
* 경험치
* 개체 등급
* 잠재 능력
* 강화 수치

목록에서는 동일한 몬스터를

```text
주황버섯 ×2
```

처럼 합산하여 표시한다.

---

## 2. 아이템

모든 아이템은 Inventory 시스템으로 관리한다.

현재 구현

* 스타터 소환권

예정

* 일반 소환권
* 특별 소환권
* 이벤트 소환권
* 소비 아이템
* 강화 아이템
* 퀘스트 아이템

모든 아이템은 Item Key 기반으로 관리하여
데이터 추가만으로 확장할 수 있도록 설계한다.

---

## 3. 뽑기 시스템

플레이어는 소환권을 사용하여
랜덤 몬스터를 획득한다.

현재 구현

* 스타터 소환권
* 랜덤 몬스터 획득

예정

* 일반 소환권
* 특별 소환권
* 이벤트 소환권
* 확률 시스템
* 연속 뽑기

플레이어 명령어는

```text
/뽑기
```

를 사용한다.

---

## 4. 시너지 시스템

모든 몬스터는 두 가지 특성을 가진다.

### 지역 특성

예시

* 헤네시스
* 페리온
* 엘리니아
* 커닝시티
* 슬리피우드

### 태생 특성

예시

* 버섯
* 돼지
* 슬라임
* 스톤
* 언데드

플레이어는 두 특성을 조합하여
다양한 덱을 구성한다.

# 게임 철학

* 높은 등급보다 좋은 개체가 중요하다.
* 희귀 몬스터보다 좋은 덱이 중요하다.
* 모든 몬스터는 활용 가능해야 한다.

현재는 개체값 시스템을 구현하지 않는다.

확장 가능한 구조를 유지한 상태에서
필요 시 추가한다.

---

# 튜토리얼

튜토리얼은 단계별 퀘스트 형태로 진행한다.

첫 완료 보상으로

* 스타터 소환권

을 지급한다.

튜토리얼 완료 이후에는

Guide / 미션 형태로 다시 진행할 수 있으며,

추가 보상을 획득할 수 있도록 설계한다.

게임 플레이를 강요하지 않고,

자연스럽게 명령어와 시스템을 익히도록 유도하는 것을 목표로 한다.
